This case study chronicles a strategic redesign born from my direct experience as a user of the OnStream app. Every friction point was a personal frustration, leading to highly empathetic and effective design solutions.

The Premise

OnStream is a streaming platform offering movies and TV shows with HD quality across multiple devices. While the foundation was solid, the user experience was riddled with friction points.

As a frequent user myself, I didn't need much user research as I was the user. Every confusing flow, every redundant button, every missing piece of information was a personal frustration that became a design opportunity.

My Philosophy: "Think like a surgeon, not a demolition expert". Rather than completely restructuring the app, I focused on strategic improvements that enhance without overwhelming, preserving the app's character while eliminating friction.

Strategic Solutions

Design decisions driven by real user frustrations and tested through personal use

Authentication Maze

First impressions matter. The original onboarding was a masterclass in how NOT to welcome users - confusing CTAs created immediate friction. Critical authentication flows suffered from redundancy.

The Problem

I repeatedly encountered navigation paths that led to identical destinations. For instance, both "Continue with email" and "Continue with Google" funneled to the same login screen, causing unnecessary decision fatigue.

Both "Continue with email" and "Continue with Google" led to same destination, creating a redundancy trap.
→

The Solution

Replaced the redundant "Continue with..." buttons with a direct "Create Account" path, including a clear toggle for existing users to log in.

abcdefghijklmnopqrstuvwxyz

Enhanced overlay visibility, centered content for focus, direct "Create Account" button led directly to registration, prominent skip button.
Eliminated unnecessary steps and user confusion

Home Screen Chaos

The main hub was a navigation nightmare - 5 tabs with redundant paths, and duplicate content options.

The Problem

The 5-item bottom navigation was overcrowded, with a redundant "More" tab leading to the same Profile page. Duplicate content sections (e.g., "trending" and "popular" carousels) further cluttered the interface and wasted my time.

  • 5-item bottom tab navigation
  • "More" tab led to same Profile destination
  • Duplicate content sections 'Trending' and 'Popular' created confusion
5-tab bottom navigation where "profile" and "more" led to the same page. Carousel had "trending" and "popular" options that duplicated hero section choices. Pure redundancy chaos!
→

The Solution

Removed the redundant 'More' tab and streamlined navigation to create clear, single-purpose pathways. Each navigation element now has a distinct function and destination.

  • Reduced to 4 essential navigation items
  • Eliminated redundant "More" tab completely
  • Eliminated duplicate carousel options
Floating bottom nav with 4 essential items (home, movie, series, watchlist). Removed duplicate 'trending' and 'popular' options.
Improved navigation clarity and reduced cognitive load by removing unnecessary options.

Completion Status

Critical information, such as a series' completion status was not available in the original app. As an entertainment enthusiast, I often encountered series with no clear distinction of their completion status, leading to frustrating waits for episodes that might never come.

The Problem

Critical information, such as a series' completion status was consistently missing. This forced me to make uninformed viewing decisions, leading to frustration with abandoned series.

Missing essential data like series completion status for informed viewing decisions.
→

The Solution

Added crucial information like "ongoing" or "completed" status for TV series directly on the home screen. This empowers users to make informed viewing decisions at a glance.

Added ongoing/completed status for TV series - context users actually need!
Eliminated frustration by providing essential context, preventing users from starting incomplete series unintentionally.

Movie Mafia

When users click on a movie, they're in research mode. They want to know if it's worth their time. Scattered information across multiple tabs creates unnecessary friction between curiosity and commitment.

The Problem

Critical information like release year and country of origin was completely missing, while redundant cast sections created visual clutter without adding value.

  • Missing release context and origin data
  • No country information
  • Duplicate cast sections across screens
Missing essential metadata, redundant information sections, poor content organization, and uninformed user decisions.
→

The Solution

Redesigned information architecture to consolidate redundant sections while surfacing essential metadata that enables quick, informed viewing decisions.

  • Integrated cast information into main overview
  • Added release year and country of origin
  • Eliminated redundant content sections
Complete essential metadata, consolidated information design, improved content hierarchy, and faster decision-making.
Faster content evaluation and selection process

Series Page

Series pages are more complex than movies. Users need episode access, series context, and related recommendations, that help users make informed viewing decisions.

The Problem

Same as Movie information, critical information was buried while redundant details cluttered the interface. Cast information appeared in multiple places, and important context like release year and country was missing.

  • Missing release context and origin data
  • No country information
  • Duplicate cast sections across screens
  • Unnecessary redundant "episode list" button for series
Separate "casts" section when cast information was already present in overview. Missing release year and country left users without important contextual information for content discovery.
→

The Solution

Restructured information hierarchy to eliminate redundancy. Removed duplicate cast sections and episode lists where information was already available. Added valuable context while maintaining clean visual design.

  • Integrated cast info in overview
  • Added release year for temporal context
  • Included country for global awareness
  • Removed redundant episode list button
Consolidated cast information within overview section. Added release year and country to provide essential context. Overview now includes: description, released year, genre, casts, country, production.
Streamlined content pages, empowered users with critical data points for quicker and more confident viewing decisions.

The Free Figma Hustle

Working within free tier limitations while designing a complex streaming platform meant I had to be creative with prototyping workarounds and component management. Cause let's be real, not everyone has enterprise budgets for learning projects, so I had to work really slyly.

Key Learning
Constraints breed creativity. Limited resources forced me to think more strategically about component systems and prototype efficiency.

Maintaining Complex Features

Maintaining robust search and filtering while simplifying the interface was like solving a puzzle. Search, filters, and watchlists required extra UX planning within design constraints.

Key Learning
Simplicity isn't about removing features – it's about making complex functionality feel effortless through thoughtful information architecture.

Evolution vs Revolution

Modernizing while preserving user familiarity was more nuanced than expected. I enhanced familiar patterns rather than replacing them, respecting user investment in the current system.

Key Learning
Revolutionary results can come from evolutionary changes. Small, thoughtful improvements often have bigger impact than dramatic overhauls.

What This Project Taught Me

Key takeaways that demonstrate strategic thinking and readiness for UI/UX roles, born from the unique "user-as-designer" perspective.

Industry Readiness

This proved I can identify real problems, think through solutions systematically, and ship work that actually improves things, regardless of experience level.
